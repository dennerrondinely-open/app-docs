### `LoanDocument` e `Document` (Servicing)

Arquivos:

- `lib/libs/servicing/models/loan_document_model.dart`
- `lib/libs/servicing/models/document.dart`

`LoanDocument` é usado no endpoint `GET /v1/servicing/loan/all`:

```json
{
  "uuid": "loan-uuid",
  "instalmentValue": 250.0,
  "instalmentNumber": 12,
  "grantedAt": "2025-10-10T00:00:00.000Z",
  "disbursements": []
}
```

`Document` é usado no endpoint `GET /v1/servicing/documents/{loanUuid}`:

```json
{
  "url": "https://.../file.pdf",
  "date": "2026-02-01T00:00:00.000Z",
  "type": "CCB"
}
```

## Relação modelo x camada

A leitura recomendada é:

1. `models/` para entender contrato de dados
2. `api/` para entender endpoint de origem
3. `repositories/` para transformação e regras
4. `bloc/state` para consumo em UI

## Boas práticas para evolução

- manter os modelos `JsonSerializable` como fonte da verdade
- evitar duplicar contratos em `modules`
- criar `copyWith` e `Equatable` quando o modelo participa de estado de tela
- manter enums e `fromJson` centralizados no domínio
